# Autorizacao forte para trabalho Codex no Godot

Este arquivo registra a intencao do usuario: Codex deve agir com autonomia alta dentro deste projeto Godot.

Codex pode, quando necessario para desenvolver ou corrigir o jogo:

- rodar validacoes headless do Godot;
- rodar validacoes do editor em headless;
- executar o jogo em janela grafica quando solicitado;
- ler logs e saidas completas do terminal;
- corrigir scripts, shaders, cenas e configuracoes;
- criar sistemas, recursos, materiais e documentos de suporte;
- remover apenas arquivos temporarios criados pelo proprio Codex dentro do workspace.

Limites importantes:

- este arquivo nao substitui as permissoes reais do sandbox Codex;
- comandos que abrem GUI, escrevem fora do workspace ou precisam de acesso externo ainda podem exigir aprovacao pela ferramenta;
- a regra final e sempre validar pelo terminal antes de declarar a tarefa concluida.
